# FocusMCP — marketplace

> **Official catalog of atomic MCP bricks for FocusMCP.**
>
> [focusmcp.dev](https://focusmcp.dev) · [PRD](./PRD.md) · [Core](https://github.com/focus-mcp/core) · [Client](https://github.com/focus-mcp/client)

This repository is the **third pillar** of FocusMCP (after `core` = the TypeScript library and `client` = the Tauri app). It contains:

- a **pnpm monorepo** where each `bricks/<name>/` folder is an atomic MCP brick versioned independently;
- a **JSON catalog** (`catalog.json`) automatically generated by CI from each brick's `mcp-brick.json` manifest and from an `external_bricks.json` file that references externally-hosted bricks;
- a **JSON Schema** (`schemas/catalog/v1.json`) published on GitHub Pages that describes the catalog format.

## Status

Active development — pre-MVP. See [PRD.md](./PRD.md).

## Layout

```
bricks/              — official bricks (each subfolder = 1 workspace package)
  <name>/
    mcp-brick.json   — manifest (name, description, tools, deps…)
    package.json     — version, name `@focus-mcp/<name>`
    src/             — brick source
    ...
external_bricks.json — externally-hosted bricks referenced in the catalog
external_bricks/     — (optional) related assets
schemas/catalog/     — catalog JSON Schema (v1.json)
schemas/examples/    — valid / invalid samples for tests
scripts/             — catalog generator (build-catalog.ts)
config/              — commitlint, lint-staged, vitest, gitleaks
.github/             — CI, release, templates, renovate
.focusmcp/           — generated output (catalog.json, not committed)
```

## Commands

```bash
nvm use
pnpm install
pnpm lint            # Biome
pnpm typecheck
pnpm test            # Vitest (generator + bricks)
pnpm build:catalog   # builds dist/catalog.json and validates it against the schema
```

## Versioning

Each brick has **its own version** (Changesets in `independent` mode). Tags follow the `@focus-mcp/<name>@<version>` pattern (e.g. `@focus-mcp/indexer@0.2.1`). No package is published to npm at MVP: distribution happens via **GitHub Releases** (tarballs) and the `catalog.json` served from GitHub Pages.

## Submitting a brick

1. Open a "Brick submission" issue describing your proposal (domain, motivation, atomicity, license).
2. Once accepted, open a PR that adds `bricks/<name>/` (or an entry in `external_bricks.json` if the brick stays outside this repo).
3. Follow [CONTRIBUTING.md](./CONTRIBUTING.md): bare-domain naming (no `focus-` prefix), atomicity rule (1 brick = 1 domain), tests, manifest, SPDX.
